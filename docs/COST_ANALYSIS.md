# HN每日新闻推送系统 - 成本效益分析

> 📅 分析日期：2026-03-11  
> 👤 分析部门：户部  
> 📊 版本：v1.0

## 一、项目概况

### 1.1 核心需求
- **数据源**：91个HN热门博客RSS源
- **更新频率**：每日北京时间8:00（UTC 0:00）
- **输出内容**：每篇文章100-200字AI摘要 + 原始链接
- **交付方式**：GitHub Pages静态网页
- **目标用户**：程序员群体

### 1.2 技术架构
```
RSS源 → Python后端 → AI摘要 → MD文件 → GitHub Pages
         ↓
    GitHub Actions
    (定时任务)
```

## 二、成本评估

### 2.1 AI API成本（核心成本）

#### 方案对比

| 方案 | 模型 | 单次成本 | 日成本 | 月成本 | 优缺点 |
|------|------|---------|--------|--------|--------|
| **A. OpenAI GPT-4o-mini** | gpt-4o-mini | $0.15/1M tokens | **$0.45** | **$13.5** | ✅ 质量高<br>❌ 成本高 |
| **B. OpenAI GPT-3.5-turbo** | gpt-3.5-turbo | $0.5/1M tokens | **$1.50** | **$45** | ✅ 速度快<br>❌ 质量一般 |
| **C. 国产模型（智谱GLM）** | glm-4-flash | ¥0.001/千tokens | **¥0.90** | **¥27** | ✅ 成本低<br>✅ 中文优化 |
| **D. DeepSeek V3** | deepseek-chat | ¥0.001/千tokens | **¥0.27** | **¥8.1** | ✅ 性价比高<br>✅ 质量接近GPT-4 |
| **E. 本地模型（Ollama）** | llama3.2 | $0 | **$0** | **$0** | ✅ 完全免费<br>❌ 需要服务器 |

#### 成本计算依据
- **假设每日新增文章**：30-50篇（保守估计）
- **每篇文章摘要长度**：150字 × 1.5 tokens ≈ 225 tokens
- **Prompt + 输入内容**：500 tokens
- **单篇文章总tokens**：725 tokens
- **每日总tokens**：50篇 × 725 = 36,250 tokens

**推荐方案：DeepSeek V3**
- 成本：¥0.27/天 = ¥8.1/月 ≈ $1.12/月
- 质量：接近GPT-4水平
- 中文优化：原生支持中文

## 三、ROI分析

### 3.1 价值评估

#### 直接价值
- **时间节省**：用户无需浏览91个博客，节省30-60分钟/天
- **信息质量**：AI摘要提取核心内容，提高阅读效率
- **持续更新**：自动化流程，无需人工维护

#### 间接价值
- **品牌建设**：技术影响力提升
- **流量导入**：可作为其他产品的入口
- **社区建设**：聚集程序员群体

## 四、风险与缓解措施

### 4.1 成本风险

| 风险 | 影响 | 概率 | 缓解措施 |
|------|------|------|----------|
| RSS源数量激增 | API成本上升 | 中 | 设置每日文章上限（50篇） |
| AI API涨价 | 成本上升 | 低 | 多模型备选方案 |
| GitHub Actions额度用尽 | 服务中断 | 低 | 迁移到云函数（阿里云函数计算 ¥0.00011167/次） |

### 4.2 技术风险

| 风险 | 影响 | 概率 | 缓解措施 |
|------|------|------|----------|
| AI摘要质量差 | 用户体验差 | 中 | 多轮测试 + 人工审核初期 |
| RSS源失效 | 内容缺失 | 高 | 定期检查 + 自动移除失效源 |
| 网页访问慢 | 用户体验差 | 低 | CDN加速 + 静态资源优化 |
